# Game Translator

一套**通用、不遠端注入、不易被防毒誤報**的日文遊戲翻譯工具。用「遊戲自己載入」或「檔案預翻」的方式把日文遊戲翻成中文，避開 MTool 那種會被掃毒的 DLL 注入。

- **翻譯引擎可插拔**：現成字典（離線）、DeepL、或**本地 Ollama LLM**（離線、無審查、能翻 H；可用 galgame 專用的 **Sakura** 模型）。
- **繁體/簡體可選**（OpenCC 台灣在地化）。
- **翻過的自動累積成可沿用 JSON**，還有**全域共用字典**跨遊戲加速。
- **翻譯監控面板**：批次翻譯時即時顯示進度條/ETA/速度 + GPU 溫度/使用率/顯存。
- **記住上次的遊戲、字典與模型**：重開工具自動還原上次選的遊戲、字典與 Ollama 模型，檔案對話框回到上次資料夾。
- **內建健檢/防呆**：開跑前先驗 Ollama 服務與模型名對不對得上、失敗跳窗+寫 log、部署失敗自動回滾、連續失敗自動熔斷；另有「掃描/測試模型」一鍵驗哪顆能用。

---

## 支援的遊戲類型（當前）

| 引擎 | 判定依據 | 翻譯方式 |
|---|---|---|
| **RPG Maker MV** | `www/js/rpg_core.js` | 塞 JS plugin（遊戲自己載入）+ 底層畫字 hook 即時查表 |
| **RPG Maker MZ** | `js/rmmz_core.js` | 同上（注入點改在 `main.js` 前） |
| **加密 RPG Maker MZ** | `data/*.json` 為 `{uid,bid,data}` 密文 | 部署時 Python 解密（自動爆破金鑰）→ 抽字 → Sakura 批次預翻建完整離線字典 → 離線 hook 查表；`data` 檔零改動 |
| **TyranoScript（Electron 打包）** | `resources/app.asar` 內含 `.ks` | 解包 asar → 批次預翻 `.ks`（含回想/畫廊/選單資料陣列文字、**標籤屬性顯示文字** `text`/`hint`/`label`）→ 改名 asar 讓遊戲改用翻好的資料夾 |

> 選遊戲主程式後**自動判型**（加密 MZ 會另標「（加密）」）。Unity / 原生遊戲目前未支援（見「規劃中」的 OCR 兜底）。
>
> 加密 MZ 目前針對 `bid 1.8.1` 加密器家族（同款被許多同人 MZ 採用，金鑰自動偵測）。解密已對真實遊戲驗證；整款端到端翻譯覆蓋率需實機驗收。

## 安裝與需求

### 方式 A：打包好的 exe（免裝 Python）
到 `dist/GameTranslator/`，雙擊 **`GameTranslator.exe`**（onedir、無主控台黑窗）。整個資料夾要一起搬。

### 方式 B：原始碼
需要 Python **3.10 ～ 3.12**（`PySide6 6.7` 目前沒有 Python 3.13+ 的預編譯 wheel，用 3.13 以上會裝不起來）：
```
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```
> 同時裝了多個 Python 版本的人，把第一行改成指定 3.12：`py -3.12 -m venv .venv`（避免 `python` 抓到系統預設的 3.13+）。
啟動：雙擊 **`啟動翻譯工具.bat`**，或 `.\.venv\Scripts\python.exe main.py`。

## 翻譯來源（可混用）

| 來源 | 誰在翻 | 特性 |
|---|---|---|
| **現成字典 JSON** | 別人/MTool 早翻好的（`翻译文件.json`、`AI翻译文件.json` 等） | 離線、免費、秒出；命中率看字典完整度 |
| **DeepL API** | DeepL 伺服器 | 需 API key；會拒譯/消毒露骨內容 |
| **本地 Ollama** | 你電腦上的本地 LLM | **離線、無審查、能翻 H**；建議有 NVIDIA 顯卡 |

**推薦組合**：現成字典打底（命中即免費、秒出）＋ **本地 Ollama（Sakura）** 補字典沒有的（含 H）。翻過的自動存回快取，重玩不用再翻。

## 本地 Ollama 設定

### 1. 裝 Ollama
<https://ollama.com/download> → Windows → 裝 `OllamaSetup.exe`（開源、正常應用程式、不注入，不會被掃毒）。

### 2. 選模型
- **Sakura（galgame 專用，最推薦）**：專為日文 galgame/輕小說翻譯微調，保留對話引號、eroge 語氣、控制碼，明顯優於通用模型。設定見下方 3。
- **通用備選**：`ollama pull qwen2.5:14b`（約 9GB，需約 10GB VRAM；顯卡小用 `qwen2.5:7b`）。

### 3. 匯入 Sakura（galgame 專用）
兩種裝法，**擇一即可**。裝完務必看下方 ⚠️「模型名要對上」。

#### 方法 A（最簡單，推薦）：一行 `ollama pull` 直抓
```
ollama pull hf.co/SakuraLLM/Sakura-GalTransl-7B-v3.7:IQ4_XS
```
（連不上 HuggingFace 就把 `hf.co` 換成 `hf-mirror.com`。）

裝完模型在 Ollama 裡的名字，就是**上面那一整串** `hf.co/SakuraLLM/Sakura-GalTransl-7B-v3.7:IQ4_XS`，**不是** `sakura`。
所以在工具裡要：按「**重新整理**」→ 模型下拉框**選那一整串**（不要留在預設的 `sakura`）→ 再按翻譯並開啟。
> 名字裡有 "sakura"，工具會自動套用 Sakura 專用提示詞，翻譯品質一樣。

#### 方法 B：Modelfile 匯入（多帶調好的參數，輸出更穩）
1. 到 HuggingFace 下載 GGUF（約 4GB）：`SakuraLLM/Sakura-GalTransl-7B-v3.7` → `Sakura-Galtransl-7B-v3.7-IQ4_XS.gguf`（連不上把 `huggingface.co` 換 `hf-mirror.com`）。
2. 在 GGUF 同資料夾建 `Modelfile`：
   ```
   FROM ./Sakura-Galtransl-7B-v3.7-IQ4_XS.gguf
   PARAMETER temperature 0.1
   PARAMETER top_p 0.3
   PARAMETER repeat_penalty 1
   PARAMETER num_predict 512
   SYSTEM 你是一个轻小说翻译模型，可以流畅通顺地以日本轻小说的风格将日文翻译成简体中文，并联系上下文正确使用人称代词，不擅自添加原文中没有的代词。
   ```
3. 匯入：`ollama create sakura -f Modelfile`
4. 之後工具的**模型欄填 `sakura`**（工具會自動用 Sakura 專用提示詞）。

> ⚠️ **模型名要對上（最常見的坑）**：工具是拿模型下拉框裡的**那串文字**去叫 Ollama，名字不完全一致（大小寫、`hf.co/...` 整串都算）就會對不上。工具會在**開始翻譯前先擋下並跳窗提示**，不會白改遊戲檔（詳見下方「疑難排解」）。
> - 用**方法 A**：按「重新整理」→ 下拉**選** `hf.co/SakuraLLM/...` 那一整串，別留 `sakura`。
> - 想繼續用預設 `sakura`：幫方法 A 抓來的模型取別名 → `ollama cp hf.co/SakuraLLM/Sakura-GalTransl-7B-v3.7:IQ4_XS sakura`，之後模型欄留 `sakura` 即可。
> - 用**方法 B**：`ollama create sakura` 建出來的名字就是 `sakura`，跟預設一致。
> - 不確定 → 按工具的「**掃描/測試模型**」，或 `ollama list` 查實際裝了哪些名字。

> Sakura 輸出簡體 → 由工具的「繁體」選項用 OpenCC 轉繁，不用擔心。

### 4.（選用）模型放別的槽
設使用者環境變數 `OLLAMA_MODELS`（例如 `D:\Ollama\models`），然後**完全退出 Ollama（系統匣右鍵 → Quit）再重開**才生效。

### 5. 確認就緒
`ollama list` 或瀏覽器開 `http://127.0.0.1:11434/api/tags` 應看到模型。

## 使用流程

### RPG Maker MV / MZ
1. 開工具 → **選擇遊戲主程式**（`Game.exe`）→ 顯示判型。
2. 選翻譯來源：
   - **離線字典**：按「選擇既有字典 JSON」選遊戲附的字典，key 留空。
   - **本地 Ollama**：引擎下拉選「本地 Ollama」→ 按「**重新整理**」→ 下拉**選實際安裝的模型**（不確定就按「**掃描/測試模型**」看哪顆綠燈可用）。工具會記住你的選擇，下次重開自動沿用。
   - **DeepL**：填 API key。
3.（可搭配上面的翻譯選項）按 **開始** → 自動部署 + 直接啟動遊戲（不經 inject.exe）。首次遇到的文字即時翻、之後走快取。

### 加密 RPG Maker MZ
1. 選遊戲主程式 → 顯示「偵測到：RPG Maker MZ（加密）」。
2. 引擎選「本地 Ollama」、模型 `sakura`（加密遊戲拿不到現成完整字典，靠引擎預翻補齊）。
3. 按 **開始** → **背景解密全部 `data` → 抽字 → Sakura 批次預翻**（跳出翻譯監控面板顯示進度）→ 翻完自動啟動。
   - `data/*.json` **全程不改**：遊戲執行期自己解密進記憶體，我方只在部署時讀取用來建字典；離線 hook 於畫字當下查表替換。
   - 遊戲若附現成 `翻译文件.json`(MTool) 會自動併入當底，已翻的不重翻。

### TyranoScript
1. 選遊戲主程式 → 顯示「偵測到：TyranoScript」。
2. 選翻譯來源（同上）。
3. 按 **開始** → **背景批次預翻所有劇本** → 翻完自動啟動中文版。
   - 過程會跳出**翻譯監控面板**：進度條、已翻/總段數、速度、ETA、GPU 溫度/使用率/顯存（長批次跑一晚時很實用）。

> **從 0 全翻**：不選字典、引擎選 Ollama，即可把整款遊戲從頭翻（沒有現成字典打底時較久，但**單句失敗會重試/跳過、每批邊翻邊存**，可放著跑一整晚、中途出錯不白費）。

## 疑難排解（本地 Ollama 常見問題）

工具內建多層防呆，遇到問題時**通常會直接跳對話框告訴你原因**，並把每次錯誤寫進一份 log。下面對照症狀找解法。

### 先看這個：`translator.log`
所有「啟動失敗 / 部署失敗 / 掃描結果」都會寫進 **`C:\Users\<你>\.game_translator\translator.log`**（一行一筆）。
卡關時**先打開它、或直接把整個檔貼給協助者**，比截圖更快定位。

### 一鍵自檢：「掃描/測試模型」按鈕
引擎選「本地 Ollama」後，模型列右邊有 **「掃描/測試模型」**。按下去會**逐顆實測**每個已安裝模型能不能真的翻，回報：
- 🟢 **可用**（附一句試翻樣本）／🔴 **不可用**（附原因）
- 並建議你在模型欄選哪一顆。

> 這步會實際載入模型、較慢（首顆尤其久），屬於「事前驗證」；平常按「開始」翻譯時工具只做**秒級**檢查（見下），不會每次都載模型。

### 症狀 1：按開始後跳「模型名稱對不上」
**原因**：你選的模型名，跟 Ollama 裡實際裝的名字不完全一致（大小寫、`hf.co/...` 整串都要一模一樣）。這是最常見的坑。
工具在**部署遊戲、開遊戲之前**就先擋下（不會像舊版那樣改了遊戲檔、開了遊戲才發現整片 404）。
**解法**（任一）：
1. 按「**重新整理**」，從下拉**點選**實際安裝的名稱（不要用手打，避免多空白/錯字）。
2. 用 `ollama list` 看實際名字，一字不差貼進模型欄。
3. 幫它取個好記的別名：`ollama cp "ollama list 顯示的那串" sakura`，之後模型欄用 `sakura`。
4. 不確定哪顆能用 → 按「**掃描/測試模型**」。

### 症狀 2：跳「連不上 Ollama 服務」
**原因**：Ollama 沒開（或被關掉）。工具靠 `http://127.0.0.1:11434` 連本機 Ollama。
**解法**：確認**工作列右下角有 Ollama 圖示**；沒有就重開 Ollama。瀏覽器開 `http://127.0.0.1:11434/api/tags` 能看到 JSON 才算就緒。

### 症狀 3：翻一翻整批中止、說「連續 N 句翻譯失敗」
**原因**：這是**熔斷保護**——連續多句都失敗（多半是模型名對不上或服務中途掛掉），工具就**主動中止整批**，不再空跑數小時、也不再噴滿屏相同錯誤。訊息會直接點出是「名字對不上」還是「服務沒開」。
**解法**：照它給的提示處理（同症狀 1 / 2）。

### 症狀 4：部署失敗 / 遊戲開不起來
**原因**：極少數遊戲改過 `index.html` 的載入方式，或資料夾不是標準結構。
工具已做兩層防護：
- **注入放寬**：找不到標準 `js/plugins.js`／`main.js` 載入點時，會退而注入 `</head>`／`</body>`。
- **失敗自動回滾**：部署若中途出錯，會把已改的檔**自動還原**，不會把遊戲留在「改一半」的壞狀態。
失敗時會**跳對話框顯示原因**，並寫進 `translator.log`。
**解法**：把對話框訊息或 `translator.log` 內容貼出來即可定位；若顯示「找不到 plugins.js」，代表選到的可能不是標準 RPG Maker 資料夾。

### 症狀 5：遊戲能開但沒翻／畫面日文照舊
先確認狀態列是「已啟動…翻譯服務執行中」而非任何失敗訊息。RPG Maker 是**執行期即時翻**：首次遇到的文字會停頓一下（首句最久），之後走快取變快。若完全沒動靜，多半是前面某個症狀被擋下了——回頭看對話框/log。

## 產出的可沿用 JSON

每款遊戲的翻譯累積在 `<遊戲>/translator_dict.json`（從你選的字典開始、越翻越完整），格式與 MTool 字典互通，可保留/分享/下次直接載入。另有全域共用字典跨遊戲累積。

## 還原（移除翻譯修改）

- **RPG Maker**：GUI 按「還原遊戲」；或把 `Game.exe` 拖到 **`還原遊戲.bat`**。
- **TyranoScript**：GUI 選遊戲 → 按「還原遊戲」（刪掉解包的 `app/`、把 `app.asar` 改回來）。
- 工具部署前會**自動備份**原始檔（`.trbak`），可完整還原。

## 為什麼「不易被防毒誤報」

- **不做遠端注入**：不用 `CreateRemoteThread` / `inject.exe` 那種被防毒標記的手法。
- RPG Maker：adapter 是**遊戲自己載入的 JS plugin**（正常行為）。
- TyranoScript：純**檔案預翻**，連執行期都不掛任何東西。
- 本地 Ollama 也是正常開源應用、不注入。
- 直接啟動遊戲本體，不經任何注入器。

> 註：若要對外發佈打包的 exe，仍建議買 code signing 憑證簽章以徹底免除誤報。

## 運作原理（簡述）

- **Python 大腦**：引擎偵測、可插拔翻譯引擎（DeepL / Ollama〔含 Sakura 模式〕/ Null）、`Pipeline`（分層查詢：遊戲私有字典 → 全域字典 → 引擎；分批邊翻邊存；繁體 postprocess）、快取字典、本機伺服器（RPG Maker 執行期用）。
- **遊戲端**：
  - RPG Maker：一支 JS plugin，hook `Bitmap.drawText` 與 `convertEscapeCharacters`，對整份嵌入的字典即時查表替換（含前綴正規化）。
  - TyranoScript：解包 `app.asar`，只翻 `.ks` 的純文字行（跳過 `[標籤]`、`#名字`、`;`/`//` 註解，保留行內 `[n2]` 與結尾 `[p]`），改名 asar 讓 Electron 改用翻好的 `app/`。

詳細架構見 `docs/superpowers/specs/2026-07-04-architecture.md`。
