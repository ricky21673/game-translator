## 2. 未完成事項（依重要性）

### 2.1 P2 OCR 通用兜底 ★最大缺口
**用途**：處理「拿不到原文」的情況——Unity、原生 Win32、**加密遊戲**、以及**烤進 CG 圖片裡的文字**。這些引擎/情況無法用檔案預翻或 JS hook，只能讀螢幕。

**技術選型**（若要整合進本工具）：
- OCR 引擎：**manga-ocr**（日文遊戲最準，本地 GPU）或 **PaddleOCR**（偵測+辨識一套包辦）。
- 截圖：`mss` 或 `dxcam`。
- 翻譯：**復用現有 `Pipeline`/引擎**（Sakura 等），不用重做。
- 疊字：PySide6 透明置頂視窗。

**替代方案**：現成的 **LunaTranslator**（獨立程式，OCR+翻譯+疊字全包，可接 Sakura）——若只要 OCR 結果，直接用它比自己造快。本工具的價值在「乾淨原文」路線，OCR 是不同領域。

**可靠度誠實話**：乾淨對話框 OCR ~90%+，文字疊 CG/小字/花體則差。屬「看得懂劇情」等級，非乾淨翻譯。

### 2.2 加密 JS 遊戲 ✅ 已完成（改道：部署時檔案解密，非執行期 hook）
**問題實例**：`ゆうべは大変おたのしみでしたね。`（加密 RPG Maker MZ）。`data/*.json` 為 `{uid,bid,data}` 密文。

**實測翻案**：原以為有「特製訊息系統／字元替換」第二層，實測後確認**只有一層檔案加密**——先前疑似的亂碼是 Windows cp950 主控台顯示 UTF-8 日文的假警報。破解檔案加密後即乾淨日文，**不需要**執行期 hook。

**做法（已實作）**：偵測 `{uid,bid,data}` → Python 移植的可逆解密（自動爆破金鑰 `_K`，涵蓋 `bid 1.8.1` 家族）→ 抽字（訊息分組、控制碼保護）→ 現有 Pipeline(Sakura) 批次預翻建完整離線字典 → 沿用離線 hook 查表替換。執行期靠遊戲自解密，**`data` 檔零改動**。詳見 `docs/superpowers/specs/2026-07-04-encrypted-mz-decrypt-design.md` 與 `docs/superpowers/plans/2026-07-04-encrypted-mz-decrypt.md`。

**狀態**：程式碼完成、單元測試全綠、解密已對真實遊戲驗證（skipif smoke 實解出日文）、已打包進 exe。**端到端實機驗收（整款 Sakura 翻完、遊戲內顯示中文、量測命中率）待執行**——這步只能人工實機跑（鐵則 #7）。已知風險：抽取 key 與執行期 hook 字串吻合度需實機量測（已用該遊戲 MZ 核心碼核實 `convertEscapeCharacters` 收到整塊訊息、與抽取 `\n` 合併 key 對齊）。

### 2.3 RPG Maker 也接監控面板
目前**段級進度只接了 Tyrano**（`segment_progress` signal）。RPG Maker 走 server 執行期即時翻譯、無離散批次，監控面板只顯示 GPU、進度停在「尚未開始」。若要，可顯示「已快取句數」之類的替代指標。

### 2.4 打包 exe 的 code signing
`GameTranslator.exe` 未簽章。要對外發佈、徹底免除防毒誤報，需買 code signing 憑證（或 SignPath 開源免費方案，因 repo 已公開）。見對話中討論。

## 3. 已知限制（小、可接受，記錄備查）

- **OpenCC s2twp 偶爾過度在地化**：如 `选择项目 → 選擇專案`（選單語境下「專案」語意錯）。GalGame 容錯高；在意可改用 `s2t`（只轉字不動詞彙）。
- **Sakura 人稱代詞會猜錯**：日文不標性別，偶爾把女角用「他」。galgame 容錯範圍。
- **Tyrano 資料陣列的 `\"` 跳脫引號未處理**：`extract_segments`/`apply_translations` 用簡單 `"[^"]*"`，字串內含跳脫引號會斷。目前遊戲少見。
- **全域共用字典無 GUI 管理**：不能在介面清空/改路徑，也無淘汰機制（存 `~/.game_translator/global_dict.json`，手動刪即可重來）。
- **MV/MZ 離線整字典嵌入不含全域字典**：離線嵌入模式只用遊戲私有 cache；全域字典的加速主要惠及引擎翻譯與 Tyrano。
- **GPU 監控**：固定取 index 0 第一張卡，且讀的是整機 GPU 使用率（非本行程專屬）。
- **加密 MZ**：見 2.2。

## 5. 環境 / 怎麼跑 / 測 / 打包

- **環境**：Windows；`.venv`（Python 3.10）；相依見 `requirements.txt`（PySide6、requests、opencc、nvidia-ml-py、pytest…）。
- **跑**：`.\.venv\Scripts\python.exe main.py`，或雙擊 `啟動翻譯工具.bat`。
- **測**：`.\.venv\Scripts\python.exe -m pytest -q`（現 152 passed；`test_server.py` 偶有 Windows socket flaky，單獨重跑即綠）。
- **打包**：`.\.venv\Scripts\python.exe -m PyInstaller GameTranslator.spec`（onedir、免主控台黑窗）。
- **本地 Ollama/Sakura**：見 README「本地 Ollama 設定」。模型存 `D:\Ollama\models`（`OLLAMA_MODELS` 環境變數）。
